# Phase 6 — Source Your Own Loading UI

**Goal:** Find, install, and integrate a loading component from the npm ecosystem.

---

## What to Do

- Search npmjs.com for a spinner, skeleton, or loading library yourself — `react-loading-skeleton`, `react-spinners`, and others all exist
- Read the library's documentation and integrate it into your phase pages
- Add a simulated delay (`setTimeout`) to see the loading state in action
- The choice of library is yours — **justify it in your PR description**

---

## Why This Matters

Evaluating and integrating third-party packages is a daily engineering task.
Reading unfamiliar docs and getting something working is the real skill here, not the spinner itself.
Writing why you chose a library in the PR description mirrors real team decision-making.

---

## Commit Message

```
feat: add loading states using [chosen library]
```

Replace `[chosen library]` with the actual library name you picked.

---

## PR Rules

Phases 5-12: Tag a teammate as reviewer. They must approve before merge.
